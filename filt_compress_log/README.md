## 版本V1.1

### log_filt_compress_1.1.py
- 实现功能：引入多进程编程，处理效率提高，亿数量级的log只需一个星期即可筛选完毕
- 问题一：压缩log未实现多进程
- 问题二：各个进程结束时间不确定，导致主进程无法继续运行后面脚本
- 问题三：进程数量未进行控制，导致部分CPU性能比较低的服务器会超频，特别是Windows Server 2008基本不可以正常工作
